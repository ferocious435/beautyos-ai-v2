# Skill Routing Workflow

## Step 1: Detect Task Type
Determine the type of work from the list below.

## Step 2: Select Skills
Use the routing table — pick primary (global) skill first, project skill only if unique:

| Task Type | Global Skill | Project Skill |
|-----------|-------------|--------------|
| Architecture | `senior-architect` | — |
| Frontend / UI | `frontend-ui-dark-ts`, `frontend-dev-guidelines` | — |
| Bot | `telegram-bot-builder` | — |
| Mini App | `telegram-mini-app` | — |
| Content / Posts | `content-creator` | `copywriting`, `social-content` |
| Image AI | `ai-studio-image` | `ai-studio-image` (ref) |
| Instagram | `instagram-automation` | — |
| Marketing | — | `marketing-ideas`, `marketing-psychology` |
| Brainstorming | `brainstorming` | — |
| Security | `security-audit` | — |
| Testing | `test-driven-development` | — |
| Product | `product-manager` | — |
| Orchestration | `loki-mode` | — |

## Step 3: Read & Apply
- Read `SKILL.md` of selected skill(s) before starting
- Follow its instructions during execution

## Step 4: Report
After completion, note which skills were used and their effect.
