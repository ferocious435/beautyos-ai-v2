# Пакет рабочих процессов аудита безопасности

## Обзор

Комплексный рабочий процесс для аудита безопасности веб-приложений, API и инфраструктуры. Этот пакет объединяет навыки для тестирования на проникновение, оценки уязвимостей, сканирования безопасности и устранения рисков.

## Когда использовать этот процесс

Используйте этот процесс при:
- Проведении аудита безопасности веб-приложений.
- Тестировании безопасности API.
- Проведении тестов на проникновение (Pentest).
- Сканировании на наличие уязвимостей.
- Укреплении (Hardening) безопасности приложения.
- Соответствии стандартам безопасности.

## Фазы рабочего процесса

### Фаза 1: Разведка (Reconnaissance)
1. Определение целевой области.
2. Сбор информации.
3. Картирование поверхности атаки.
4. Идентификация используемых технологий.

**Используемые навыки**: `@scanning-tools`, `@shodan-reconnaissance`.

### Фаза 2: Сканирование уязвимостей
1. Запуск автоматизированных сканеров.
2. Проведение статического анализа (SAST).
3. Сканирование зависимостей.
4. Выявление ошибок конфигурации.

**Используемые навыки**: `@vulnerability-scanner`, `@security-scanning-security-sast`, `@security-scanning-security-dependencies`.

### Фаза 3: Тестирование веб-приложений (OWASP Top 10)
Тестирование на:
- Внедрение SQL (SQL Injection).
- Межсайтовый скриптинг (XSS).
- Нарушение аутентификации.
- Небезопасные прямые ссылки на объекты (IDOR).
- Обход путей файлов (Path Traversal).

**Используемые навыки**: `@sql-injection-testing`, `@xss-html-injection`, `@broken-authentication`, `@idor-testing`.

### Фаза 4: Тестирование безопасности API
1. Перечисление эндпоинтов API.
2. Тестирование аутентификации и авторизации.
3. Тестирование ограничений скорости (Rate Limiting).
4. Проверка валидации входных данных.

**Используемые навыки**: `@api-fuzzing-bug-bounty`, `@api-security-best-practices`.
